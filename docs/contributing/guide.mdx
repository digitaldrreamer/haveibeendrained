## Getting Started

We welcome contributions! Here's how to get started:

<Steps>
<Step title="Fork the repository">

Fork the repository on GitHub and clone your fork:

```bash
git clone https://github.com/YOUR_USERNAME/haveibeendrained.git
cd haveibeendrained
```

</Step>

<Step title="Set up development environment">

Follow the [Local Setup](/contributing/local-setup) guide to get your environment running.

</Step>

<Step title="Create a branch">

Create a feature branch:

```bash
git checkout -b feature/your-feature-name
```

</Step>

<Step title="Make your changes">

Make your changes and test them locally.

</Step>

<Step title="Submit a pull request">

Push your changes and open a pull request on GitHub.

</Step>
</Steps>

## Contribution Guidelines

### Code Style

- **TypeScript/JavaScript:** Follow existing code style
- **Rust:** Follow Rust formatting standards (`cargo fmt`)
- **Commit messages:** Use clear, descriptive messages

### Testing

- Write tests for new features
- Ensure all existing tests pass
- Test edge cases and error conditions

### Documentation

- Update documentation for new features
- Add code comments for complex logic
- Update README if needed

## Areas for Contribution

### High Priority

- **Bug fixes** - Fix reported issues
- **Documentation** - Improve docs
- **Tests** - Increase test coverage
- **Performance** - Optimize slow operations

### Features

- **Detection algorithms** - Improve threat detection
- **UI/UX** - Enhance user experience
- **API features** - Add new endpoints
- **Widget improvements** - Enhance embeddable widget

## Pull Request Process

1. **Update documentation** if needed
2. **Add tests** for new features
3. **Ensure tests pass** locally
4. **Update CHANGELOG.md** with your changes
5. **Submit PR** with clear description

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers
- Focus on constructive feedback
- Help others learn and grow
